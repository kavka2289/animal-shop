

# Документация для научного руководителя

## 1. Описание проекта

Тема курсовой работы: веб-приложение «магазин для животных».

Технологии: язык Python, фреймворк Django, контейнеризация Docker.

На текущем этапе реализована клиентская часть интерфейса: макет страниц интернет-магазина с навигацией и демонстрационным набором товаров. Данные о товарах и категориях хранятся в виде констант в модуле `shop/data.py`.
Структура проекта:

- `pet_store` — настройки проекта Django, маршрутизация корня.
- `shop` — приложение витрины: представления, шаблоны, статические файлы (CSS), тестовые данные.

## 2. Пользовательские сценарии

Сценарии описывают поведение пользователя в текущей версии (только просмотр и переходы).

**Сценарий 1. Просмотр главной страницы**

1. Пользователь открывает адрес сайта.
2. Отображается главная страница с кратким описанием, блоком категорий и блоком «Популярное».
3. Пользователь может перейти в каталог или в любую категорию.

**Сценарий 2. Просмотр каталога и фильтрация по категории**

1. Пользователь открывает раздел «Каталог» (или переходит по ссылке с главной).
2. Отображается список всех демонстрационных товаров.
3. Пользователь выбирает категорию в боковом списке (собаки, кошки, птицы, грызуны).
4. Отображаются только товары выбранной категории.

**Сценарий 3. Просмотр карточки товара**

1. Пользователь нажимает на товар в каталоге или на главной.
2. Открывается страница товара с названием, кратким описанием и ценой.
3. Внизу показываются связанные товары из той же категории (если есть).

**Сценарий 4. Информационные страницы**

1. Пользователь открывает «О магазине» или «Контакты» через меню в шапке сайта.
2. Отображается соответствующий текст (учебный, без реальной обработки форм обратной связи).

**Сценарий 5. Ошибка «страница не найдена»**

1. Пользователь вводит несуществующий адрес в пределах сайта.
2. Отображается страница с кодом 404 и ссылками в каталог или на главную (при выключенном режиме отладки Django; в режиме отладки возможна стандартная страница ошибок Django).

## 3. Глоссарий терминов

| Термин | Краткое пояснение |
|--------|-------------------|
| Django | Веб-фреймворк на Python: маршруты URL, шаблоны, ORM, админка и др. |
| Шаблон (template) | HTML-файл с вставками синтаксиса Django для вывода данных и общих блоков (наследование `extends`). |
| Представление (view) | Функция или класс, обрабатывающий HTTP-запрос и возвращающая ответ (часто рендер шаблона). |
| URLconf | Сопоставление путей URL с представлениями. |
| Статические файлы | CSS, JS, изображения, отдаваемые как файлы без исполнения на сервере. |
| Статика (collectstatic) | Команда Django, собирающая статические файлы в каталог `STATIC_ROOT` для продакшена. |
| WhiteNoise | Middleware для отдачи статики из того же процесса, что и приложение (удобно в Docker). |
| Docker | Платформа контейнеризации: образ с зависимостями и изолированным запуском приложения. |
| docker-compose | Файл описания сервисов (здесь один сервис `web`) для сборки и запуска контейнера. |
| Gunicorn | WSGI-сервер для запуска Django в продакшене вместо встроенного `runserver`. |
| Витрина | Публичная часть магазина: каталог и карточки товаров для посетителя. |
| Категория товара | Группировка товаров (в проекте: собаки, кошки, птицы, грызуны). |
| Макет / демонстрационные данные | Нефиктивный набор товаров и цен для показа интерфейса без реальной коммерции. |
